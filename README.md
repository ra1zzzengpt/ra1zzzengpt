Everything I build lives in the **terminal**. No GUI, no electron, no nonsense —
just clean CLI interfaces, readable architecture, and code that's easy to extend.
Right now I'm leveling up by shipping small but real projects: terminal messengers,
password utilities, file inspectors, HTTP clients, sorting visualizers.

## 💎 &nbsp; What I care about in code

```diff
+ Clear project structure
+ Readable names, simple logic
+ Separation: UI / API / business logic
+ Safer input handling
+ Useful console output
+ Real projects over toy examples
+ Gradual refactoring over rewrites
- Cargo-cult patterns
- Premature abstractions
- "It works on my machine"
```
